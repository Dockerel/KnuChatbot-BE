## 1. 마이페이지 조회 개선
### 개요
* 유저 마이페이지 화면에서 질문 수를 필요로 함

### 문제 및 의사결정 과정
* 마이페이지에서 회원별 질문 수 조회 시, 매번 전체 질문 테이블을 집계하는 쿼리 발생
```sql
SELECT COUNT(*)
FROM Member m
JOIN m.histories h
JOIN h.questions q
WHERE m.id = :memberId
```
* 다중 테이블 JOIN 및 COUNT 쿼리가 발생하여 마이페이지 조회 속도가 느려지고 DB에 부하 누적
* 프론트 팀원과 회의한 결과 질문 수는 단순한 UI 표시용 지표이며, 사용자의 거래나 중요한 작업에는 영향을 주지 않는 부가 정보임을 확인
* 이를 바탕으로 다음 내용을 제안
  1. 비즈니스 임팩트가 낮은 questionCount 데이터를 member 테이블에 반정규화하여 별도 저장
  2. 스케줄링 작업을 통해 매일 저장된 값과 실제 질문 개수가 일치하는지 정기적으로 검증

### 성과
* 마이페이지 조회 속도를 1.6초에서 132ms로 약 90% 개선하면서도, 스케줄링 된 검증 작업을 통해 반정규화의 데이터 무결성을 보장
* 조회 시에는 단순 SELECT로 빠르게 처리하면서도, 배치 검증으로 데이터 정합성이라는 반정규화의 위험을 완벽하게 관리할 수 있음
* 좋은 개발자는 단순히 기술 문제를 기술로 해결하는 것이 아니라, 비즈니스 요구사항을 정확히 파악하고 최적의 트레이드오프를 판단할 수 있어야 한다는 것을 깨달음
